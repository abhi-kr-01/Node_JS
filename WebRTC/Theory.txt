## WebRTC: Web real-time communication -> it is browser technology 

[a-> server -> B] -> webRTC -> [A <-> B] 
- pear to pear direct communication [P2P] 

- WebRTC (Web Real-Time Communication) is an open-source technology that allows web browsers and mobile apps to 
communicate with each other directly in real-time. It enables audio, video, and data sharing between devices without 
needing to download plug-ins or route everything through central servers

- WebRTC shifts this by establishing a Peer-to-Peer (P2P) connection
 1. Direct streams:  Audio and video streams are sent directly from device to device, reducing latency and avoiding heavy server loads
 2. signaling:  Although it is P2P, a server is initially required to help devices introduce themselves, negotiate connection settings, and establish network paths (a process known as NAT traversal).
 3. Security:  WebRTC mandates encryption, making secure communication a built-in feature

- Capabilities: file transfer & screen sharing, in-Browser Video and voice and data Transfer

#concept used in WebRTC:

1. Media Captures:
 - getting access of camera , mic ,screen 
 - Browser API : navigator.mediaDevices.getUserMedia()

2. p2p communication :
 - Object in JS: RTCPeerConnection
 - provide browser <-> browser connection 
 - handles audio packets, video packets and network optimisation

3. Signaling:
 - WebRTC doesn't provide signaling 
 - webRtc only handles media transfer not finding user
 - we need our own signaling system (socket.io or WebSocket)
 - signaling : [A <-> B] data exchaing throuogh server without long polling
 - it is reuire only on setup phase 
 -

4. SDP(Session Description Protocol):
 - browser says:
    I support:
    video: h264
    audio: opus
    resolution: 720p 
 - this info stored in SDP (resume of browser media capabilties)
 FLOW: [A caller OFFER -> server(socket.io) -> B receive OFFER, B create ANSWER -> server -> A receive ANSWER] : offer/Answer model

5. ICE Candidates: 
 - Users are behind routers.

 -WebRTC tries many possible routes.
    These routes are: ICE candidates 

 - browser says: 
    Try this IP
        No?
        Try this one
        No?
        Try relay server
    this is called ICE candidates exchanges
