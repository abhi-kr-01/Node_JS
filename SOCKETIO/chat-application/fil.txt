The io.on() method is the primary way to listen for events on a Socket.IO server instance. It is an event listener that triggers a callback function whenever a specific event occurs across the entire server or a specific namespace

// Server-side example
io.on('connection', (socket) => {
  console.log('A user connected with ID:', socket.id);

  // You can then listen for events on that specific socket
  socket.on('disconnect', () => {
    console.log('User disconnected');
  });
});

##
Why can’t backend ask frontend for username during disconnect?

Because:

disconnect

means the connection is already gone.

Think:

Phone call ended.

You can’t suddenly ask:

“Wait, what was your name?”

No connection exists anymore.

That’s why server must already store:

socket.id → username

before disconnect happens.

Very important backend architecture concept.
