4. HASH

Redis Hash is similar to MongoDB document.

Store:

await redis.hset(
    "user:1",
    "name",
    "Abhi"
);

Multiple fields:

await redis.hset(
    "user:1",
    {
        name: "Abhi",
        age: 22,
        city: "Patna"
    }
);

Redis:

user:1

name -> Abhi
age  -> 22
city -> Patna

Get one field:

const name = await redis.hget(
    "user:1",
    "name"
);

Get entire hash:

const user = await redis.hgetall(
    "user:1"
);

Output:

{
    name:"Abhi",
    age:"22",
    city:"Patna"
}
5. LIST

Like arrays.

Push right:

await redis.rpush(
    "tasks",
    "Learn Redis"
);

await redis.rpush(
    "tasks",
    "Learn Docker"
);

Redis:

tasks

[
 Learn Redis,
 Learn Docker
]

Get list:

await redis.lrange(
    "tasks",
    0,
    -1
);

Output:

[
 "Learn Redis",
 "Learn Docker"
]

Remove first item:

await redis.lpop("tasks");

Used in:

Queues
Background Jobs
Notifications
6. SET

Unique values only.

Store:

await redis.sadd(
    "skills",
    "NodeJS"
);

await redis.sadd(
    "skills",
    "Redis"
);

await redis.sadd(
    "skills",
    "Redis"
);

Redis:

NodeJS
Redis

Only one Redis.

Get all:

await redis.smembers(
    "skills"
);

Check member:

await redis.sismember(
    "skills",
    "Redis"
);

Output:

1

Used in:

Followers
Tags
Permissions
Roles
7. Sorted Set (ZSET)

Stores score + value.

Example:

await redis.zadd(
    "leaderboard",
    100,
    "Abhi"
);

await redis.zadd(
    "leaderboard",
    300,
    "Rahul"
);

await redis.zadd(
    "leaderboard",
    200,
    "Aman"
);

Redis:

Rahul 300
Aman  200
Abhi  100

Get ranking:

await redis.zrevrange(
    "leaderboard",
    0,
    2
);

Output:

[
 "Rahul",
 "Aman",
 "Abhi"
]

Used in:

Leaderboards
Game Scores
Trending Posts
Rankings
8. Pub/Sub

Redis can act like a message broker.

Publisher:

await redis.publish(
    "chat",
    "Hello"
);

Subscriber:

redis.subscribe("chat");

redis.on("message",
(channel,message)=>{
    console.log(message);
});

Used in:

Chat Apps
Notifications
Real-time Events
Methods You Must Master First

Phase 2 (Objects)

publish()

Redis Streams

These are the concepts used in real-world backend systems and interviews.

For your current Redis setup project, I'd recommend starting with String operations + TTL + caching MongoDB data, because that's where Redis provides immediate practical value.
